## 9) Integrations & Webhooks — real infrastructure

Currently `/admin/integrations` is an empty stub and the `Add Integration` form only covers outbound configuration. Build a proper integration/webhook backbone.

### 9a) Integration Types expanded
Add `Webhook` to the Integration Type dropdown alongside existing `Payment, Email, Storage, CRM, Analytics, Automation`.

### 9b) Incoming Webhooks (external systems → Academy)
For each integration of type `Webhook` (or for specific named providers like `meetup`, `stripe`, `kommo`, `sendgrid`), auto-generate a stable endpoint:

```
POST https://academy.findandstudy.com/api/integrations/{provider}/webhook
```

For each endpoint:
- **Signing secret**: generated via `Generate new secret` button; stored server-side (env: `ACADEMY_WEBHOOK_SECRET_{PROVIDER}` if set, otherwise in `integrations.signing_secret_encrypted`). Show last-4 and `last_rotated_at`. Include a `Rotate` and `Revoke` action.
- **HMAC verification**: require `X-Signature` (hex-encoded HMAC-SHA256 of raw body using signing secret) and `X-Timestamp` (unix seconds; reject if |now - ts| > 300). Return 401 on mismatch.
- **Idempotency**: honor `Idempotency-Key` header; store last 1000 keys per provider and return cached response on repeat.
- **Event subscriptions**: admin selects which events this integration emits (e.g., `booking.created, booking.updated, booking.cancelled`). Unknown events are 202-accepted but logged.
- **Event → Action mapping**: a rule editor to say "when `booking.created`, upsert user by email then enroll in course X and send welcome email". Store as JSON rules; apply on webhook receipt.
- **IP allowlist** (optional) per integration.
- **Rate limit**: 60 req/min per IP, 600 req/min per integration (configurable).

### 9c) API Keys (external systems → Academy REST)
New section `Admin > Integrations > API Keys`:
- Create, list, rotate, revoke keys.
- Each key has `name, scopes (users:read, users:write, enrollments:write, content:read, chat:invoke), expires_at, ip_allowlist, last_used_at`.
- Secret shown **once** at creation only.
- Middleware validates `Authorization: Bearer <key>` and enforces scopes per route.

### 9d) Outgoing Webhooks (Academy → external systems)
For events like `user.created, enrollment.created, certificate.issued, course.completed`:
- Admin configures target URL, signing secret, subscribed events, active toggle.
- Academy POSTs signed payloads with `X-Academy-Signature` + `X-Academy-Timestamp`.
- Retry with exponential backoff (1m, 5m, 30m, 2h, 12h — 5 attempts total). Persist every attempt.

### 9e) Event Log (central)
`Admin > Integrations > Event Log` table: timestamp, direction (in/out), provider, event type, HTTP status, signature valid?, idempotency key, duration, error. Row click → payload (masked secrets) + response body + **Replay** button.

### 9f) Test tools
Per integration: `Test Connection` (for API/SMTP types) and `Send Test Event` (for webhook types, sends a synthetic payload to the target or to the Academy handler).

### 9g) Environment/Secrets status UI
Show status (not values!) of expected env vars: `ACADEMY_API_URL`, `ACADEMY_API_KEY`, `ACADEMY_WEBHOOK_SECRET`, `MEETUP_*`, `STRIPE_*`, etc. Each shows `configured | not configured | last updated`. Provide an inline help panel explaining how to set them in Replit Secrets and restart the server.

### 9h) Public Integration Docs
A read-only page `/admin/integrations/docs` (also linkable externally) per provider describing: endpoint URL, required headers (`X-Signature`, `X-Timestamp`, `Idempotency-Key`, `Content-Type`), payload schema, cURL example, signature generation examples in Node.js and Python.

### 9i) Provider presets
Ship presets for `MeetUp`, `Stripe`, `Kommo`, `SendGrid`, `Generic Webhook`. Selecting a preset prefills name, type, default endpoint path, default event list, and docs links.
